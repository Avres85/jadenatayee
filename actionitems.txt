Action Items: Realistic Page Flip + Touch Gestures (Single Static PDF)

Scope lock (important)
- Project supports one static portfolio PDF only (~10MB).
- No upload pipeline, no multi-document library, no dynamic source switching.
- Use StPageFlip path only.
- Drop all custom flip-engine/mounting tracks.

Checklist
1. Architecture and scope
- [x] Lock engine to StPageFlip (no custom implementation branch).
- [x] Keep current static source model (`public/portfolio-compressed.pdf` + `/api/viewer/document`).
- [x] Keep existing toolbar controls (Prev/Next, Jump, Zoom, Fullscreen).

2. Flip realism core
- [ ] Replace current CSS spread animation (`flip-next`/`flip-prev`) with real page curl interactions.
- [ ] Enable corner grab, drag-follow, and release-to-complete.
- [ ] Implement drop cancel + snap-back animation when thresholds are not met.
- [ ] Ensure page turn is page-level (not whole spread transform).

3. Touch and gesture behavior
- [ ] Add robust touch/pointer handling for mobile and tablet.
- [ ] Support swipe fallback for quick next/prev.
- [ ] Add gesture arbitration so pinch/pan does not trigger flip.
- [ ] Prioritize pan when zoom > 100%; require explicit corner grab to flip.

4. Rendering and smoothness
- [ ] Integrate PDF page render output cleanly with the flipbook view.
- [ ] Preload adjacent pages for smoother turns.
- [ ] Keep white page backing for transparent PDF content.
- [ ] Preserve odd/even spread pairing consistency.

5. Reliability and fallback
- [ ] Handle pointer cancel/interruption without stuck flip states.
- [ ] Block concurrent flips and prevent page index desync.
- [ ] Respect reduced-motion with simplified transition fallback.
- [ ] Keep black stage and no persistent center crease at rest.

6. QA gates
- [ ] Test desktop: Chrome, Safari, Firefox.
- [ ] Test mobile touch: iOS Safari, Android Chrome.
- [ ] Validate grab/drag/drop realism and snap-back behavior.
- [ ] Validate zoom+pan+flip arbitration under fast repeated input.
